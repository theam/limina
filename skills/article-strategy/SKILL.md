# Article Strategy

Turn `kb/` research into high-value technical articles.

## Workflow

### Phase 1: Proposal

1. Read `kb/mission/CHALLENGE.md`.
2. Read `kb/ACTIVE.md`.
3. Read the relevant findings, experiments, reviews, and lesson files.
4. Propose 5-8 article ideas with:
   - title
   - angle
   - target audience
   - source files in `kb/`
5. Ask the user which ideas to write.

### Phase 2: Writing

For each selected article:

1. Read all cited source files.
2. Write the draft to `kb/articles/ART{NUM}-slug.md`.
3. Include front matter with title, date, status (`DRAFT`), and sources.

## Quality Rules

- No fluff.
- Make claims traceable to `kb/` sources.
- Include failures and trade-offs, not only wins.
- Favor actionable takeaways for practitioners.
